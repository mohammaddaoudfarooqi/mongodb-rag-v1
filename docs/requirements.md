# Requirements ‚Äî Hybrid Search RAG with MongoDB

A Streamlit RAG app that answers customer-support questions grounded in a seed
knowledge base, retrieved via rank fusion of `$vectorSearch` and `$search` on
MongoDB Atlas, generated by Claude Haiku 4.5 on Bedrock.

## Functional requirements

### Ingestion

- **REQ-001** ‚Äî THE SYSTEM SHALL load `knowledge_base.json` from the repo root,
  one document per chunk, no splitting.
- **REQ-002** ‚Äî THE SYSTEM SHALL embed each chunk with the document model
  (`voyage-4`, 1024 dims) and store the vector in `embedding`.
- **REQ-003** ‚Äî THE SYSTEM SHALL upsert chunks into
  `knowledge_base.kb_chunks` keyed on a deterministic id (e.g. `metadata.source` +
  `title`) so re-running ingest is idempotent.
- **REQ-004** ‚Äî WHEN ingest starts THE SYSTEM SHALL ensure the vector and Atlas
  Search indexes exist and are READY before any write.

### Embeddings

- **REQ-010** ‚Äî THE SYSTEM SHALL expose a single `Embedder` protocol in
  `app/embeddings.py` with `embed_documents` and `embed_query` methods.
- **REQ-011** ‚Äî THE SYSTEM SHALL select the backend via `EMBEDDING_PROVIDER`
  (`atlas` default, `voyage` alternate) through a factory; no other module
  branches on provider.
- **REQ-012** ‚Äî THE SYSTEM SHALL embed documents with `voyage-4` and queries
  with `voyage-4-lite`.
- **REQ-013** ‚Äî IF a returned embedding does not match `EMBEDDING_DIMENSIONS`
  (default 1024) THEN THE SYSTEM SHALL raise immediately on startup / first call.

### Indexes

- **REQ-020** ‚Äî THE SYSTEM SHALL ensure a vector index on `embedding` with
  cosine similarity and `numDimensions = EMBEDDING_DIMENSIONS`.
- **REQ-021** ‚Äî THE SYSTEM SHALL ensure an Atlas Search index over `text` and
  `title` with the standard analyzer.
- **REQ-022** ‚Äî WHEN the app starts THE SYSTEM SHALL poll until both indexes
  report status `READY` before serving queries, displaying a
  *"Preparing search indexes‚Ä¶"* spinner in the UI.

### Retrieval

- **REQ-030** ‚Äî THE SYSTEM SHALL retrieve top-k chunks (default `k = 5`) using
  hybrid rank fusion of `$vectorSearch` and `$search`.
- **REQ-031** ‚Äî THE SYSTEM SHALL apply weights `HYBRID_VECTOR_WEIGHT = 0.7` and
  `HYBRID_LEXICAL_WEIGHT = 0.3` (vector-leaning, env-configurable).
- **REQ-032** ‚Äî WHEN `db.command("buildInfo").version` is `>= 8.1` THE SYSTEM
  SHALL use `$rankFusion`; OTHERWISE the system shall use `$unionWith` +
  reciprocal-rank fusion.
- **REQ-033** ‚Äî WHEN the app starts THE SYSTEM SHALL log exactly one line
  naming the chosen retrieval path: `retrieval: rankFusion` or
  `retrieval: unionWith+rrf`.
- **REQ-034** ‚Äî THE SYSTEM SHALL expose a `Retriever` protocol with signature
  `retrieve(query: str, k: int) -> list[Chunk]` where each `Chunk` carries
  `title`, `text`, `metadata`, and a fusion `score`.

### Generation

- **REQ-040** ‚Äî THE SYSTEM SHALL stream an answer from Claude Haiku 4.5 on
  Bedrock (`us.anthropic.claude-haiku-4-5-20251001-v1:0`, region `us-east-1`)
  via `boto3` `bedrock-runtime` ConverseStream ‚Äî not the Anthropic SDK.
- **REQ-041** ‚Äî THE SYSTEM SHALL enable Bedrock prompt caching on the system
  prompt.
- **REQ-042** ‚Äî THE SYSTEM SHALL use a system prompt that instructs the model
  to *answer only from the provided context; otherwise say "I don't have that
  information."*
- **REQ-043** ‚Äî IF the retrieved context does not support an answer THEN THE
  SYSTEM SHALL emit the exact canned response *"I don't have that
  information."*
- **REQ-044** ‚Äî THE SYSTEM SHALL expose an `LLMClient` protocol with
  `stream(system: str, messages: list[dict]) -> Iterator[str]`.

### UI

- **REQ-050** ‚Äî THE SYSTEM SHALL present a single text input plus Submit
  button.
- **REQ-051** ‚Äî WHEN the user submits a question THE SYSTEM SHALL stream the
  answer token-by-token into the page.
- **REQ-052** ‚Äî THE SYSTEM SHALL render an expandable "Sources" panel listing,
  per chunk: `title`, `metadata.source`, fusion score.
- **REQ-053** ‚Äî THE SYSTEM SHALL accept `Retriever` and `LLMClient` via a
  `get_app(retriever, llm)` factory (or constructor args) so tests can inject
  fakes.

### Configuration / safety

- **REQ-060** ‚Äî THE SYSTEM SHALL load configuration from `.env` via
  `python-dotenv` (or `uv run --env-file .env`).
- **REQ-061** ‚Äî THE SYSTEM SHALL maintain a committed `.env.sample` listing
  every variable with no real values.
- **REQ-062** ‚Äî THE SYSTEM SHALL gitignore `.env` and `.mcp.json`.
- **REQ-063** ‚Äî THE SYSTEM SHALL never echo secrets in logs, errors, or the
  UI.
- **REQ-064** ‚Äî THE SYSTEM SHALL be import-side-effect-free: no module-level
  `boto3.client(...)`, `MongoClient(...)`, or index bootstrap; those run
  inside `main()` / `get_app()`.

### Observability

- **REQ-070** ‚Äî THE SYSTEM SHALL log the chosen retrieval path on startup
  (see REQ-033).

## Non-functional requirements

- **REQ-NF-001** ‚Äî Tests SHALL run with no live Atlas, Bedrock, or Voyage
  credentials.
- **REQ-NF-002** ‚Äî CI (GitHub Actions) SHALL run ruff and pytest on push.
- **REQ-NF-003** ‚Äî Test coverage SHALL include the golden-path question
  (returns) citing `policies/returns.pdf`, and the fallback path returning the
  exact canned response.

## Unchanged / non-goals

- Auth, multi-tenant, conversation memory, document upload UI ‚Äî out of scope.

## Premortem

| # | Failure mode | Mitigation (EARS) |
| - | --- | --- |
| 1 | Embedding model returns 1536 / 768 dims; vector index expects 1024; queries silently fail or never match. | THE SYSTEM SHALL verify the dimension of one document and one query embedding on startup and raise a clear error on mismatch (REQ-013). |
| 2 | Ingest writes happen before Atlas Search index is READY; first queries return zero lexical hits, masked by vector recall. | WHEN ingest starts THE SYSTEM SHALL poll both indexes to READY before any write (REQ-004, REQ-022). |
| 3 | M0 cluster runs MongoDB 8.0; code uses `$rankFusion` (8.1+) and aggregation errors at runtime instead of at startup. | WHEN the app starts THE SYSTEM SHALL detect server version and pick the `$unionWith`+RRF path on < 8.1 (REQ-032), logged once (REQ-033). |
| 4 | A test imports `app.main` and triggers a real `MongoClient(...)` / `boto3.client(...)` ‚Äî CI fails or hits live infra. | THE SYSTEM SHALL keep `import app` side-effect free; clients are constructed inside `main()` / `get_app()` (REQ-064). Enforced by a unit test that imports the package with no env vars set. |
| 5 | LLM hallucinates an answer when retrieval returns empty / off-topic chunks. | THE SYSTEM SHALL use a system prompt that mandates the *"I don't have that information."* canned reply when context is insufficient (REQ-042, REQ-043), with a fallback-path test that asserts the exact phrase (REQ-NF-003). |
